OBS Events — does it hang together as a real platform?
======================================================

A code-first audit of every journey and the connections between them.

**Date** 2026-07-10**Method** code-first trace · 80 agents**Findings** 62 verified · **0** refuted**Scope** review only — no fixes**7**Blockers**31**Major**24**Minor**62**Total defects

Verdict
-------

**Not yet.** The _engine_ is genuinely strong — money in integer paise, webhook-sourced fulfilment, clean role guards, idempotent crons, and a door-ready ticket → QR → validation path any platform would be happy to ship. But the _product layer_ on top is riddled with dead-ends, fakes, and one-way links that a discerning user, a paying sponsor, or a first-time organizer hits within minutes. Today it reads as a polished shell over an unfinished product.

The three things hurting it most:

1.  **The signature features have no data path.** No event editor — organizer or admin — can attach a speaker, link a 100 Days program day, or flag a launch, so Speakers, the 100 Days Program, and the Launchpad are structurally empty in production; admins can’t even create a program or publish an article. [F17](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F17), [F25](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F25), [F35](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F35)
    
2.  **The primary discovery surfaces are fake.** The global header search and the /webinars & /summits pages render a hardcoded mock catalog whose every card and suggestion dead-ends on a non-existent slug. [F1](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F1), [F2](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F2)
    
3.  **Core promises are dishonest.** Paid online-event attendees are told a join link exists but nothing delivers it; password reset 404s; the homepage brags “1,500+ events” above empty rails; sponsors are sold header/hero/VIP placements that don’t exist. [F6](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F6), [G1](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#G1), [F56](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F56), [F23](https://6890a98b-791d-43ea-93bf-d263990b09dd.frame.claudeusercontent.com/_f/1783688528-e2e6/#F23)
    

Top 10, ranked
--------------

If the team fixed only ten things to make this feel like a real platform, these, in order — ranked by how many users hit them, how core the journey is, and how badly it reads to a paying evaluator.

1.  Three signature sections — Speakers, the 100 Days Program, and the Launchpad — have no way to receive data. No event editor (organizer or admin) can attach a speaker, link a program day, or flag a launch, so in production those pages, plus the home rails that feed off them, are permanently empty unless someone edits MongoDB by hand. One missing UI silently kills three headline features.
    
2.  The search box in the header is on every page and is the first thing a user reaches for. It runs on a hardcoded mock catalog, so every suggestion navigates to a slug that does not exist and lands on “not found,” while real events never surface. The primary discovery control is uniformly broken.
    
3.  Two prominent nav destinations (/webinars, /summits) render an entirely fabricated catalog — fake “180K+ likes,” random “PROMOTED” badges, raw prices — and every card dead-ends on a 404. A discerning user clicks one card and immediately distrusts the whole product.
    
4.  Password reset is 100% dead: there is no “Forgot password?” link anywhere, and the reset link the server emails 404s because the SPA has no /reset-password route. Table-stakes account recovery is impossible for the primary credential type — the backend implements it fully; only the two client screens are missing.
    
5.  A user pays for an online event and is told across the UI and emails that a join link will be provided — but no shipped surface ever delivers event.meetingLink. At event time the paying attendee has a valid ticket and no way to attend. A broken core promise at the moment money changed hands.
    
6.  The 100 Days Program — the platform’s signature concept — has no admin surface at all (no page, no route); it exists only because a seed helper created it, and day themes can never be set. Articles are the same (F35). The team cannot operate two whole content pillars from the product.
    
7.  The money relationship dead-ends: approving a “Become a sponsor” lead creates no sponsor and shows nothing on /sponsors. An admin must silently re-key everything into a separate form. The terminal step of the revenue flow produces no visible outcome.
    
8.  Any user can create a chapter, but a normal user then has no path to put an event in it and nothing explains why (event creation is organizer-gated). The headline “create a chapter” action produces a dead object with no next step — the exact confusion the brief flagged, entirely unhandled.
    
9.  The homepage asserts “108 Chapters / 54 Countries / 1,500+ Events” as hardcoded fact directly above rails that visibly show no events. Fabricated marketing numbers over an empty product is the single fastest way to read as fake to a senior evaluator.
    
10.  Admin permissions have no single source of truth: the Users “Role” dropdown is decoupled from the organizer profile the guards actually check — promoting to ORGANIZER grants no access (dead-ends), and demoting to USER revokes nothing (the de-authorized user keeps full control of their events). A false sense of control over capability is worse than none.
